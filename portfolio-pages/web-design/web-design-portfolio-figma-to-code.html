project: translate Figma design to code

Created a quick tutorial on how to convert a Figma design into code. The goal was to show how easy it is to take a design and convert it to working code.

We used a simple design to show how to convert the design into code. The design was created in Figma and then converted leveraging the custom cross-platform design system and component library.

This is the minimium viable product (MVP) of the tutorial and was intneded to be my notes to build out a learning network module. Developers started using the Miro version, providing feedback that this really helped them synthasize the custom cross-platform design system and component library as a whole.

13. Button

lorem ipsum...

notes for this section:

Props

variant: button | submit | reset

button: has no default behavior, and does nothing when pressed by default. It can have client-side scripts listen to the element's events, which are triggered when the events occur. src: MDN docs

submit: submits the form data to the server. This is the default if the attribute is not specified for buttons associated with a <form>, or if the attribute is an empty or invalid value. src: MDN docs

reset: The button resets all the controls to their initial values. (This behavior tends to annoy users.) src: MDN docs

name

submitted as a pair with the button's value as part of the form data, when that button is used to submit the form src: MDN docs

value

defines the value associated with the button's name when it's submitted with the form data. This value is passed to the server in params when the form is submitted using this button src: MDN docs
